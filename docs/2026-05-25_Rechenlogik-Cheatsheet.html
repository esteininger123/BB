03Rechenformeln Schritt-für-Schritt

Aufruf-Reihenfolge in recalc(i) (Z. 580 ff). Jede Formel zeigt den exakten Output-Key, den die UI nutzt.

3.1 Investition + Eigenkapital

Kaufpreis Gesamt
kpGesamt = kaufpreis + stellplatzKp

Hard-Guard: wenn kpGesamt = 0 → recalc returnt null (Audit E-1).

Kaufnebenkosten (KNK)
knkPct = grEstPct + 0,015 (Notar) + 0,005 (Grundbuch)
knk = kpGesamt × knkPct

Keine Maklerprovision (B&B verkauft direkt). GrESt-Default 5 % (BaWü).

Investition + EK + Darlehen
investitionGesamt = kpGesamt + knk
ekBedarf = knkMitfinanziert ? 0 : knk
darlehen = knkMitfinanziert ? investitionGesamt : kpGesamt

AfA-Gutachten-Kosten sind RAUS aus EK-Bedarf (Iter 7) — B&B trägt.

3.2 Annuität + Laufzeit

Monatliche Annuität
annuityMo = darlehen × (zins + tilgung) / 12

UI-Anzeige = deutsche Formel. Für CF-Berechnung wird intern annuityRate() = Excel-PMT genutzt (~4 €/J Drift sonst, QA-Fix 22.05.).

Laufzeit in Monaten (NPER)
nper = CEILING(−LN(1 − darlehen × rateM / annuityMo) / LN(1 + rateM), 1)

Min(480, ceil). Bei 4,5 % / 1 % Tilgung ≈ 360 Mo (30 J).

3.3 AfA

AfA-Bemessung & AfA-Jahreswert
anschaffungskosten = kpGesamt + knk
afaBemessungBetrag = anschaffungskosten × gebaeudeAnteil
afaJahr = afaBemessungBetrag × afaSatz
afaMo = afaJahr / 12

KNK IST in AfA-Basis drin (Iter 79, BFH-konform). Standard 85 % Gebäude / 15 % Boden.

3.4 Mietsubvention — 2 Phasen

Backend rechnet computeAutoSubvention() (Z. 483 in [weId].js) — Frontend übernimmt das Ergebnis als subventionPhasen.

Hebel-Formel
X_ideal = MbV × ((1 + Kappung)² − 1) // 2 Erhöhungsstufen
X_max_markt = Marktmiete − MbV // rechtl. Cap
X_final = min(X_ideal, X_max_markt)
Phase 1 (Monat 1 bis 36 − monateSeit)
p1Mo = X_final
p1Monate = max(0, 36 − monateSeit)
Phase 2 (immer 36 Mo)
mieterErhP2 = min(MbV × Kappung, Marktmiete − MbV)
p2Mo = max(0, X_final − mieterErhP2)
p2Monate = 36

Effektivmiete (= MbV + X_final) bleibt für den Käufer über alle 72 Mo konstant — Mieter zahlt nach Erhöhung mehr, B&B's Anteil sinkt entsprechend (Iter 47/48 Glättung).

Cap auf Subv-Total
cap = max(5.000 €, qm × 200 €, MbV × 18)
wenn (p1Eur + p2Eur) > cap: X wird so reduziert, dass Total = cap
Mindestschwelle
wenn Total < 1.000 €: phasen = [] (keine Vermarktungs-Story)

3.5 Mietsteigerung — Sprung vs Staffel

Erste Erhöhung (Monat M1)
M1 = max(1, 36 − monateSeitMieterhoehung)
Sprung (Bestand) — alle 36 Mo, exponentiell
nSprünge(m) = m≥M1 ? floor((m−M1)/36) + 1 : 0
Faktor(m) = (1 + steigerungProz)^nSprünge(m)
Staffel (Neuvermietung) — jährlich, linear
nSprünge(m) = m≥M1 ? floor((m−M1)/12) + 1 : 0
Faktor(m) = 1 + nSprünge(m) × steigerungProz
Marktmiete-Cap (alle Modi)
cap(y) = marktmieteEurQm × qm × (1 + wertsteigerung)^(y−1)
kaltmieteForMonth(m) = min(kaltmiete × Faktor(m), cap)

Im Sprung-Modus wird der Cap am letzten Sprung-Übergang eingefroren (Iter 91.5), damit Belastung J7-J10 konstant bleibt.

3.6 Cashflow Jahr y (Schleife y = 1..30)

Mieteinnahmen
kaltmieteMo = Σ kaltmieteForMonth(m für m in y) / 12
spMieteMo = stellplatzMiete × (1 + wertsteigerung)^(y−1)
subventionMo = Σ subvForMonth(m) / 12
mieteJahr = (kaltmieteMo + spMieteMo + subventionMo) × 12
Zinsen + Tilgung (Excel CUMIPMT / CUMPRINC)
zinsenJahr = −cumipmtExcel(rateM, nper, darlehen, startP, endP)
tilgungJahr = −cumprincExcel(rateM, nper, darlehen, startP, endP)
restschuld = max(0, restschuld − tilgungJahr)
Hausgeld-Block (konstant über Laufzeit, Edgar 24.05.)
hgJahr = (hausgeld + mietverwaltung + hausverwaltung) × 12

HG-Inflation ist seit 24.05. komplett raus. Hardcode 0.

Steuervorteil (Werbungskosten-Ansatz)
wkAfa = afaBemessungBetrag × afaSatz
stVerlust = wkAfa + zinsenJahr + mvJahr + hvJahr − mieteJahr
stVorteilJahr = stVerlust × steuersatz

Mietverwaltung + Hausverwaltung gelten als nicht-umlagefähige WerbKost. Hausgeld-Rücklage ist NICHT in WerbKost (anders als manche Berater rechnen).

Cashflow Jahr nach Steuern
cfJahr = mieteJahr − zinsenJahr − tilgungJahr − hgJahr + stVorteilJahr

3.7 Vermögen Jahr 0..10

Marktwert + Verkaufserlös
startwert = (marktwertProQm > 0 ? marktwertProQm × qm : kaufpreis) + stellplatzKp
wert(y) = startwert × (1 + wertsteigerung)^y
verkaufserlös(y) = wert(y) − restschuld(y)
kumCf(y) = Σ cfJahr(1..y)
vermoegenBrutto(y) = verkaufserlös(y) + kumCf(y)
vermoegenNetto(y) = vermoegenBrutto(y) − ekBedarf

Stellplatz-Marktwert = Stellplatz-Kaufpreis (kein separater Markt-Vorteil, wächst aber mit).

3.8 IRR 10 Jahre

Cashflow-Reihe für IRR
irrSeries = [−ekBedarf, cf[0], cf[1], ..., cf[8], cf[9] + verkaufserlös(10)]
irr = newtonRaphson + Bisektion-Fallback

Wenn ekBedarf ≤ 1 € (z.B. KNK-mitfin) → IRR = null, Frontend zeigt Zuwachs-Satz statt.

3.9 Bank-Bonität (Quick-Modus)

Anrechenbare Miete (Bank-Bewertung)
bonMieteAnr = (kaltmieteForMonth(1) + stellplatzMiete + subvForMonth(1)) × 0,8

80 %-Cap = Banken-Standard (Leerstand/Ausfallrisiko). Kaltmiete wird gegen Marktmiete-Cap gekappt (FS-3a Fix 25.05.).

Bonität-Delta
bonVor = bonEinnahmen − bonAusgaben
bonDelta = bonMieteAnr − annuityMo (Quick)
= bonMieteAnr − annuityMo − hgMo − hvMo (Detail)
bonNach = bonVor + bonDelta

3.10 Brutto-Rendite (Tag-1-KPI für WE-Liste)

Bruttorendite
tag1Miete = kaltmieteForMonth(1) + stellplatzMiete
phase1Subv = subventionPhasen[0].mo (oder subventionMo)
bruttorendite = (tag1Miete + phase1Subv) × 12 / kpGesamt

3.11 Sparen vs Investieren (Vergleich-Sektion)

Nur-Sparen-Pfad (10 Jahre)
nurSparen(y) = ekBedarf × (1 + sparZins)^y
Investieren-Pfad (CF auf Tagesgeld)
tagesgeld(y) = tagesgeld(y−1) × (1 + sparZins) + cfJahr(y)
mitImmo(y) = tagesgeld(y) + (wert(y) − restschuld(y))

07Was die Engine NICHT kann

Bewusste Vereinfachungen — wenn der Käufer/Bank diese Punkte anspricht: Steuerberater / Notar / Bank weiterreichen.

Wichtigste Engine-Grenze
Die Engine ist eine Tag-1-Projektion. Sie kalkuliert was passiert, wenn alle Annahmen halten. Sie ist KEIN Stress-Test gegen Bank-Pleite, Mieterausfall, Inflations-Shock oder Notarverschiebung. Wenn der Käufer „aber was wenn..." fragt: ehrlich sagen „dazu rechne ich separat" — NICHT in der Engine improvisieren.
